## [Books](/books/)

Two books on software architecture and AI-native systems. The first, *Universal Microservices Architecture*, is published on Amazon through Apress. It covers the full UMA model across 13 runnable chapters with 100% business logic coverage enforced in CI. The second, *The Day After*, is forthcoming — it builds the organizational model for restructuring software for the age of AI agents, role by role, using C-DAD as its structural foundation.

## [White Papers](/whitepapers/)

Five research papers published between 2023 and 2025. They trace the development of the frameworks from first principles. CSMA (2023) applied service-oriented thinking to the browser. UMA (2024) extended that to every runtime in the stack. ECCA (2025) addressed event governance. C-DAD (2025) defined how AI agents can navigate software through contracts rather than code. The UMA + MCP paper (2025) covers how UMA and the Model Context Protocol combine to produce adaptive agentic systems.

## [Projects](/projects/)

Four open source projects built directly from the frameworks. UMA-code-examples provides 13 chapters of runnable Rust and WASM code from the book. the-day-after-toolkit is a CLI that audits codebases for agent-readiness and scaffolds the contracts needed to fix what it finds. Traverse is a runtime that implements C-DAD principles. youaskm3 is a knowledge layer built for agentic navigation. All are spec-governed and CI-enforced.
